Git双系统交替使用遇到的问题

发表于 2020-06-08 更新于 2020-06-07

在双系统环境下（如Linux，Windows）交替使用Git，可能会遇到一些问题。通过设置Git配置项可以解决。

  1. Linux下文件换行符号是LF，而Windows下文件换行符号是CRLF。为了避免版本库中源文件两种换行符混杂的情况，可以在Windows中设置Git配置项core.autocrlf=true。表示在检出版本库时，自动将LF转换成CRLF;在提交版本库时，自动将CRLF转换成LF。

  2. 在Windows系统中检出之前在Linux系统中提交的文件时，会发现某些文件变成了已修改的状态，实际上本地文件和版本库中的文件内容完全一样。这时候无论使用git checkout -- <file>还是git reset --hard HEAD都无法改变已修改的状态，导致无法拉取代码或者切换分支。

    比较单个文件时发现本地和版本库的文件模式不同。

    解决方法：Git有个配置项core.filemode，值为true时Git会比较文件模式，为false时忽略文件模式。只要将全局配置项设为false，并删除项目配置项即可。注意：项目配置项如果为true，会覆盖全局配置项false，造成改动不生效。

    现在本地文件和版本库状态一致了

  3. 扩展知识

    Git中平时用到的有三种配置文件：系统，全局，项目

    system: {安装目录}/etc/gitconfig

    global: {用户目录}/.gitconfig

    repository: {项目目录}/.git/config

Python使用logger打印程序运行时间

发表于 2020-06-03 更新于 2020-06-04

Python中可以通过导入logging模块和time模块打印程序运行时间

  1. logging模块

    • 导入

      import logging

    • 配置

      可以选择很多消息级别，如debug、info、warning、error以及critical。通过赋予不同的消息级别，开发者就可以只输出错误信息到特定的日志文件，或者只在调试时输出调试信息。

      basicConfig函数各参数：

      filename：指定日志文件名

      filemode：和file函数意义相同，指定日志文件的打开模式，’w’或者’a’

      format：指定输出的格式和内容

      格式 说明
      %(name)s 日志的名称
      %(asctime)s 可读时间，默认格式‘2003-07-08 16:49:45,896’，逗号之后是毫秒
      %(filename)s 文件名，pathname的一部分
      %(pathname)s 文件的全路径名称
      %(funcName)s 调用日志多对应的方法名
      %(levelname)s 日志的等级
      %(levelno)s 数字化的日志等级
      %(lineno)d 被记录日志在源码中的行数
      %(module)s 模块名
      %(msecs)d 时间中的毫秒部分
      %(process)d 进程的ID
      %(processName)s 进程的名称
      %(thread)d 线程的ID
      %(threadName)s 线程的名称
      %(relativeCreated)d 日志被创建的相对时间，以毫秒为单位

      datefmt：指定时间格式，同time.strftime()

      level：设置日志级别，默认为logging.WARNNING

      日志等级 使用范围
      FATAL 致命错误
      CRITICAL 特别糟糕的事情，如内存耗尽、磁盘空间为空，一般很少使用
      ERROR 发生错误时，如IO操作失败或者连接问题
      WARNING 发生很重要的事件，但是并不是错误时，如用户登录密码错误
      INFO 处理请求或者状态变化等日常事务
      DEBUG 调试过程中使用DEBUG等级，如算法中每个循环的中间状态

      stream：指定将日志的输出流，可以指定输出到sys.stderr，sys.stdout或者文件，默认输出到sys.stderr，当stream和filename同时指定时，stream被忽略

    • 通过logging.getLogger(name)来实例化，如果name不给定就用root。名字是以点号分割的命名方式命名的(a.b.c)。对同一个名字多次调用logging.getLogger()方法会返回同一个logger对象。这种命名方式下，后面的logger是前面logger的子logger，自动继承父logger的log信息。

      logger = logging.getLogger(__name__)

    • 程序中打印日志

      logger.debug('output debug message')

      logger.info('output info message')

      logger.warning('output warning message')

      logger.error('output error message')

      logger.exception('trackback exception')

      logging.error和logging.exception这两个方法只有一个区别, loging.error只是记录一个日志消息, 而exception在记录消息的同时, 默认会记录错误发生的traceback信息, 所以说, 如果你想记录更详细的错误信息, 最好是使用exception, 但是error方法也可以输出traceback信息, 只需要设置一个参数。

      logger.error('traceback error message', exc_info=True)

  2. time模块

    • 导入

      import time

    • 返回当前时间的时间戳(1970年纪元后经过的浮点秒数)

      secs = time.time()

    • 转换时间戳为本地时间元组。 如果seconds参数未传入，则转换当前时间。

      local = time.localtime(secs)

    • 转换时间元组为可读字符串，格式由参数format决定。如时间元组未指定，则转换localtime()返回的当前时间。

      time.strftime("%Y-%m-%d %H:%M:%S", local)

      格式 说明
      %y 两位数的年份表示（00-99）
      %Y 四位数的年份表示（000-9999）
      %m 月份（01-12）
      %d 月内中的一天（0-31）
      %H 24小时制小时数（0-23）
      %I 12小时制小时数（01-12）
      %M 分钟数（00=59）
      %S 秒（00-59）
      %a 本地简化星期名称
      %A 本地完整星期名称
      %b 本地简化的月份名称
      %B 本地完整的月份名称
      %c 本地相应的日期表示和时间表示
      %j 年内的一天（001-366）
      %p 本地A.M.或P.M.的等价符
      %U 一年中的星期数（00-53）星期天为星期的开始
      %w 星期（0-6），星期天为星期的开始
      %W 一年中的星期数（00-53）星期一为星期的开始
      %x 本地相应的日期表示
      %X 本地相应的时间表示
      %Z 当前时区的名称
      %% %号本身
